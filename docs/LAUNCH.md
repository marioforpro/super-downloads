# Launch Checklist — Super Downloads

> Everything needed before going public. Items are ordered by dependency.

---

## Gate 1: Product Ready

- [ ] App icon profesional (no Tauri defaults)
- [ ] Lógica freemium implementada y testeada
- [ ] First-run onboarding funcional
- [ ] Versión visible en la app
- [ ] About screen con info de contacto
- [ ] Error handling pulido (mensajes user-friendly)
- [ ] Build limpio sin warnings
- [ ] Testado en macOS 13+ (Ventura, Sonoma, Sequoia)

## Gate 2: Distribution Ready

- [ ] Code signing decidido e implementado (o workaround documentado)
- [ ] Sistema de updates funcional (Tauri updater o manual check)
- [ ] DMGs finales generados (Apple Silicon + Intel)
- [ ] SHA256 checksums para los DMGs
- [ ] Instrucciones de instalación claras

## Gate 3: Web Ready

- [ ] Dominio registrado y configurado
- [ ] Landing page live
- [ ] Página de pricing visible
- [ ] Botones de descarga funcionales
- [ ] Terms of Service publicados
- [ ] Privacy Policy publicada
- [ ] Disclaimer de uso responsable
- [ ] OG image + meta tags
- [ ] Analytics web instalado

## Gate 4: Billing Ready

- [ ] LemonSqueezy cuenta creada y producto configurado
- [ ] Producto "Super Downloads Pro" — €29 lifetime
- [ ] License key generation habilitado (3 activaciones por key)
- [ ] Promo code LAUNCH30 creado (30% off)
- [ ] Flujo de compra funcional (web → checkout → license key → app activation)
- [ ] License validation implementada en la app (LemonSqueezy API)
- [ ] Prueba de compra end-to-end (test mode)
- [ ] Email de confirmación de compra
- [ ] Proceso de restauración de licencia en la app

## Gate 5: Communication Ready

- [ ] Canal de soporte definido (email mínimo)
- [ ] FAQ con preguntas comunes (instalación, Gatekeeper, plataformas)
- [ ] Al menos 1 red social configurada
- [ ] Copy de lanzamiento preparado
- [ ] Screenshots de la app (3-5 capturas)
- [ ] Vídeo/GIF demo (opcional pero recomendado)

## Gate 6: Launch

- [ ] Todos los gates anteriores completados
- [ ] 3-5 beta testers han probado la versión final
- [ ] Bugs críticos resueltos
- [ ] Publicar landing
- [ ] Anunciar en redes
- [ ] Monitorizar primeras horas

---

## Post-Launch Immediate (primera semana)

- [ ] Revisar analytics (web + app si hay)
- [ ] Responder a todo el feedback
- [ ] Hotfix si hay bugs críticos
- [ ] Documentar aprendizajes
- [ ] Evaluar siguientes pasos
